Nesta etapa, aprendi a utilizar a biblioteca Pandas, uma das principais ferramentas de ciência de dados em Python. Ela é usada para ler, organizar, filtrar e analisar informações armazenadas em arquivos como CSV e Excel. Com essa biblioteca, os dados podem ser tratados de forma parecida com uma planilha, mas usando comandos de programação.
O principal conceito estudado foi o DataFrame, que representa uma tabela formada por linhas e colunas. Cada coluna guarda um tipo de informação, como idade, localidade, renda ou resposta de uma pesquisa. Já cada linha representa um registro individual. Para carregar os arquivos, aprendi a usar funções como pd.read_csv() para arquivos CSV e pd.read_excel() para arquivos do Excel.
Também aprendi que, antes de realizar cálculos, é necessário explorar os dados. Para isso, podem ser usados comandos como head(), que mostra as primeiras linhas da tabela; columns, que mostra os nomes das colunas; shape, que informa a quantidade de linhas e colunas; e info(), que ajuda a identificar tipos de dados e possíveis valores vazios.
Outra parte importante foi a limpeza dos dados. Algumas tabelas possuem valores inválidos, vazios ou códigos que representam respostas como “não declarado” e “não se aplica”. Para tratar esses casos, aprendi a utilizar recursos como dropna(), que remove valores vazios, e isin(), que verifica se um valor pertence a uma lista. Esse tratamento é importante porque evita que dados incorretos interfiram nas porcentagens, médias e outros resultados.
Também estudei os filtros. Com Pandas, é possível selecionar apenas as linhas que atendem a uma condição. Por exemplo, é possível separar registros de uma determinada região, faixa etária ou categoria. Os filtros permitem analisar grupos específicos sem alterar a tabela original. Para manter os dados principais protegidos, também aprendi a usar .copy(), que cria uma cópia da tabela antes de fazer alterações.
Por fim, aprendi a unir tabelas diferentes usando a função merge(). Essa função relaciona duas tabelas por meio de uma coluna em comum, como um código de identificação. O tipo de junção left, por exemplo, mantém todos os registros da tabela principal, mesmo quando não existe uma informação correspondente na segunda tabela.
Os principais aprendizados desta etapa foram leitura de arquivos CSV e Excel, uso de DataFrames, exploração de tabelas, limpeza de dados, filtros, cópias de segurança e união de informações com merge(). Esses conhecimentos são importantes porque permitem transformar dados brutos em informações organizadas e prontas para análise.
